# VPS Speedtest Telegram Bot 🚀

Легкий Telegram-бот на Python и `aiogram` для измерения и мониторинга скорости интернета на вашем VPS/сервере.

## Возможности
- **Два источника измерений:** Использует `speedtest-cli` и `wget` (Selectel) для независимого измерения скорости скачивания. Результаты показываются отдельно для каждого источника.
- **Ручной запуск:** Мгновенное измерение скорости скачивания, загрузки (только speedtest-cli) и пинга по команде `/start`.
- **Постоянный мониторинг:** Автоматическое тестирование каждые 10 минут с отправкой результатов и графика за 24 часа прямо в чат.
- **Графики за 24 часа:** Почасовая агрегация с отдельными кривыми для каждого источника измерений. Отправляется картинкой к каждому отчёту.
- **Постоянное хранилище:** Все измерения хранятся в компактном JSON-файле на Docker volume, данные сохраняются после перезапуска контейнера.
- **В контейнере:** Простая и удобная установка с помощью Docker и Docker Compose.

## Требования
- Установленные Docker и Docker Compose на вашем сервере.
- Токен Telegram-бота (можно получить у [@BotFather](https://t.me/BotFather)).

## Установка и запуск

1. Склонируйте репозиторий на ваш сервер:
   ```bash
   git clone <ссылка-на-ваш-репозиторий>
   cd speedtest_tg_bot
   ```

2. Создайте файл `.env` из примера и укажите токен бота:
   ```bash
   cp .env.example .env
   ```
   Затем отредактируйте `.env` и установите `BOT_TOKEN`.

3. Запустите бота в фоновом режиме:
   ```bash
   docker compose up -d --build
   ```

## Использование
Просто отправьте команду `/start` вашему боту в Telegram. Он сразу ответит, запустит оба измерения скорости и начнёт отправлять отчёты каждые 10 минут. Каждый отчёт включает текстовые результаты и график скорости за 24 часа.

## Переменные окружения

| Переменная | Описание | По умолчанию |
|---|---|---|
| `BOT_TOKEN` | Токен Telegram-бота (обязательно) | — |
| `DATA_PATH` | Путь к JSON-файлу с данными | `/data/speedtest_data.json` |

## Структура проекта

```

├── graph.py            # Генерация графиков через matplotlib
├── Dockerfile          # Многоэтапная Docker-сборка
├── docker-compose.yml  # Конфигурация Docker Compose
├── requirements.txt    # Python-зависимости
└── pyrightconfig.json  # Конфигурация LSP Pyright
```
